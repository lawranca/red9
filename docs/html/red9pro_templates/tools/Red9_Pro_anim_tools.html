Pro_Pack : tools : anim_tools¶

ProPack anim_tools module deals with the more generic animation functions, primarily UI wrapping for some of the base rig inetractions setups.

>>> from Red9.pro_pack import r9pro
>>> r9anim_tools = r9pro.r9import('r9anim_tools')

Base Animation Tools with little or no other code benefits other than UI based system calls

set_space(side, aux_slot, newspace=None, align_mode='verbose')¶

set the aux_space for the wristlock and deal with transform changes

Parameters:
  • side – side of the Arm systems we’re dealing with (‘L’ or ‘R’)
  • aux_slot – which aux_slot are we modifying here
  • newspace – if given this is the new object we’re binding too, else we use the first selected object
  • align_mode – how do we initially align the systems to the new object. * verbose - throws a ConfirmDialog to let you choose * snap_to_new - snaps the wristlock to the pivot of the new object * maintain_wrist - maintains the current wrists transforms * maintain_wristlock - maintains the current wristlock transforms * no_align - does no alignment

bake_space_to_wrist(wrist, aux_obj=None, aux_slot=None, time=False, timerange=(), step=1, smartbake=False, smartbake_ref=[])¶

bake the selected mSystem auxSpace object back to the wrist controller, auxSpace object gets the keys

Parameters:
  • wrist – wrist we’re baking too
  • aux_obj – the AuxSpace base object that will track the wrist, maintaining current offset
  • aux_slot – allows us to just pass in the aux_slot rather than the connected object
  • time – do we drop keys or not?
  • timerange – range to process over
  • step – frame step between keys if time=True
  • smartbake – if True we look at the mSystem’s current keys and only match over those keys if time enabled
  • smartbake_ref – smartbake=True if given, used as reference nodes to extract keytimes from, else we look at all nodes about to be processed which isn’t always what we want. If we still find no keytimes we revert to base range times with step given
