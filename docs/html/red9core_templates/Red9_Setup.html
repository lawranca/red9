Red9_Setup¶

This is the main startup module tha controls all aspects of the boot for both StudioPack and much of ProPack

email: rednineinfo@gmail.com

Red9 blog : http://red9-consultancy.blogspot.co.uk/ MarkJ blog: http://markj3d.blogspot.co.uk ——————————————

This is the heart of the Red9 StudioPack’s boot sequence, managing folder structures, dependencies and menuItems.

######### THIS SHOULD NOT REQUIRE ANY OF THE RED9.core modules ##########

maya_screen_mapping()¶

return the maya applications screen size mapping data. If we’re running in 4k, the screen width, the screen height, as well as the dot / pixels per inch (ppi) used by some of the ui scaling code

full 4k 150% scaling : (True, 3840, 2097, 144.0dpi) full 4k 100% scaling : (True, 3840, 2097, 96.0dpi)

return: (bool(is4k), width, height, dpi)

..note::
actual dpi return is only supported in PySide2 (2017 onwards)
maya_dpi_scaling_factor()¶

this is a bodge really, but needed to remap the UI scales to 4k setups in some of the core StudioPack UI’s. This basically uses a scaling factor based on the DPI of the screen setups. This is also used in ProPack to control some of the widget sizes so we can consistently scale icon widgets as expected.

framerate_mapping(value=None, asAPI=False, truncate_float=False)¶

single place to convert fps data between string representation, actual float(fps) and MTime.unit if no valuye is passed we return a full mapping dict {pal: (25, OpenMaya.MTime.kPALFrame), …}

Parameters:
  • value – if float or int we convert the data to timeUnit (‘pal’,’ntsc’,….) if value is a string we convert to actual fps as float
  • asAPI – if value was given and this is True we return the OpneMaya.MTime.kUnit for the value
  • truncate_float – round(fps, 3) round the float to 3 decimal places.
